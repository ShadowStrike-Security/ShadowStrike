-------------------------------

If patternChars = 65535:
    patternChars * sizeof(WCHAR) = 65535 * 2 = 131070

But this is stored in a SIZE_T parameter (on x64, this is 64-bit).
The multiplication is done as:
    (USHORT)65535 * (size_t)2 = (size_t)131070

This is NOT an overflow because sizeof(WCHAR) is a compile-time constant
and the compiler promotes to size_t.

HOWEVER: RtlCompareMemory will try to read 131070 bytes from both buffers.
If the UNICODE_STRING.Length is less than this, it's a buffer overread!
